# Wan2.2-TI2V-5B × ComfyUI 詳細安裝教學手冊

適用環境:**Ubuntu 22.04 / 24.04 / 26.04** + **x86_64 或 aarch64** + **NVIDIA RTX GPU (VRAM ≤ 12GB)**

> 三個 Ubuntu 版本與兩種 CPU 架構都已驗證:不同版本的 Python 預設(3.10/3.12/3.14)與
> `libglib2.0-0`→`t64` 改名都已自動處理,**同一支 install.sh 通用**。

本手冊比 [README.md](README.md) 更詳細,涵蓋:驅動安裝、Python 版本、逐步流程、第一支影片、疑難排解、參數調校與加速。

## 1. 前置需求

| 項目 | 需求 |
|---|---|
| OS | Ubuntu 22.04 / 24.04 / 26.04 (64-bit,x86_64 或 aarch64;同一支 install.sh 通用) |
| GPU | NVIDIA RTX (建議 ≥ 8GB,你的 ≤12GB 足夠) |
| 驅動 | NVIDIA 驅動 (建議 ≥ 550,搭配 CUDA 12.x) |
| 硬碟 | 模型約 17GB + ComfyUI/相依套件約 8GB,**預留 30GB** |
| 網路 | 下載模型需穩定連線 (約 17GB) |
| Python | 3.10–3.13 (腳本會優先挑 3.12;3.14 可能還沒 PyTorch 輪子) |

## 2. 安裝 NVIDIA 驅動

> 安裝腳本**不會**自動裝驅動 (高風險、需重開機),請先手動確認。

先檢查是否已有驅動:
```bash
nvidia-smi
```
- 若能看到 GPU 名稱與 VRAM → 已就緒,跳到第 3 步。
- 若 `command not found` 或報錯 → 安裝驅動:

```bash
sudo ubuntu-drivers autoinstall   # 自動挑選建議的專有驅動
sudo reboot                       # 一定要重開機
```
重開後再次 `nvidia-smi` 確認。出現類似 `Driver Version: 5xx.xx  CUDA Version: 12.x` 即正常。

> 不需要另外安裝整包 CUDA Toolkit。PyTorch 的 CUDA 輪子自帶執行期函式庫,只要**驅動**夠新即可。

## 3. 取得這個安裝包

把整個資料夾 (含 `setup.sh`、`install.sh`、`download_models.sh`、`start.sh`、
`README.md`、`安裝手冊.md`、`風格與參數預設.md`) 放到 Ubuntu 機器上,例如 `~/WAN2.2`。
可用 `scp`、USB、git 等方式傳過去。

```bash
cd ~/WAN2.2
chmod +x *.sh
```

## 4. 執行安裝腳本

### 最簡單:一鍵全自動

```bash
./setup.sh        # 自動跑 install.sh + download_models.sh
```

### 或分步驟

```bash
./install.sh      # 只安裝環境
```

`install.sh` 會依序做 7 件事,**每一步都先偵測、已完成就略過**:

| 步驟 | 內容 | 略過條件 |
|---|---|---|
| 0 | 檢查 OS / sudo | — |
| 1 | 安裝系統套件 (git, ffmpeg, aria2, build-essential…) | 已裝的不重裝;**某套件在此版本無候選會警告略過,不中斷** |
| 2 | 偵測 NVIDIA 驅動 | 只偵測,印出 GPU 資訊 |
| 3 | 選擇 Python | 用系統 python3 (Ubuntu 26.04 = 3.14,可直接用) |
| 4 | clone ComfyUI | 已存在則 `git pull` |
| 5 | 建立 venv 虛擬環境 | 已存在則重用 |
| 6 | 安裝 PyTorch (CUDA) | 已是 CUDA 版則略過 (依 `torch.version.cuda` 判斷,不受有無 GPU 影響) |
| 7 | 安裝 ComfyUI 相依套件 | pip 自動略過已滿足的 |

> **關於 Python**:Ubuntu 26.04 預設 Python 3.14,PyTorch cu128 已有 3.14 輪子,**直接能用**,
> 不必另裝舊版。(已在 Ubuntu 26.04 x86_64 實測通過。)

**自訂選項** (環境變數):
```bash
TORCH_CUDA=cu124 ./install.sh     # 改 CUDA 版本 (預設 cu128;舊卡可用 cu124)
COMFY_DIR=/data/ComfyUI ./install.sh   # 改安裝位置
./install.sh --with-models        # 安裝完順便下載模型
```

## 5. 下載模型

```bash
./download_models.sh
```

下載方式會**自動擇優**(都支援續傳):
1. **hf** — 官方 Hugging Face CLI,含完整性校驗,最可靠 (若 venv 內沒有會自動裝)
2. **aria2c** — 多執行緒,速度最快
3. **wget** — 最後備援

會下載 3 個檔案到正確資料夾 (用 `aria2c` 多線程續傳,沒裝就用 `wget -c`):

| 檔案 | 放置位置 | 大小 |
|---|---|---|
| `wan2.2_ti2v_5B_fp16.safetensors` | `ComfyUI/models/diffusion_models/` | ~10GB |
| `umt5_xxl_fp8_e4m3fn_scaled.safetensors` | `ComfyUI/models/text_encoders/` | ~6.7GB |
| `wan2.2_vae.safetensors` | `ComfyUI/models/vae/` | ~0.5GB |

**可重複執行**:已下載完整的檔案會略過;中斷的下次會**續傳**而非重抓。

## 7. 產生第一支影片

1. ComfyUI 介面左上 → **Workflow → Browse Templates → Video**。
2. 點 **「Wan2.2 5B TI2V」** 官方範本載入。
3. 確認三個載入節點對應正確檔案:
   - **Load Diffusion Model** → `wan2.2_ti2v_5B_fp16.safetensors`
   - **Load CLIP (type: wan)** → `umt5_xxl_fp8_e4m3fn_scaled.safetensors`
   - **Load VAE** → `wan2.2_vae.safetensors`
4. **文生影片**:在正向提示詞輸入描述 (英文效果通常較佳)。
   **圖生影片**:把起始圖接到對應的 image 輸入節點。
5. **第一次先用小設定測試** (見下表),確認跑得動再放大。
6. 點 **Queue / Run**。完成後影片在 `ComfyUI/output/`。

> 第一次跑會比較久 (要把模型載入記憶體 + 編譯)。之後同一 session 會快很多。

## 8. 參數調校

TI2V-5B 是**單一模型** (不是 14B 的高/低噪雙專家),原生為 **720p @ 24fps**。

| 參數 | 正式輸出 (720p) | 第一次測試 |
|---|---|---|
| 解析度 | 1280×704 / 704×1280 | 832×480 或 640×640 |
| 長度 (frames) | 121 (=5 秒) | 49–81 |
| FPS | 24 | 24 |
| Steps | 30 | 20 |
| CFG | 5.0 | 5.0 |
| Sampler / Scheduler | uni_pc / simple | euler / simple |
| Shift (ModelSamplingSD3) | 8.0 | 5–8 |

調參原則:**一次只改一個變數**,觀察效果再動下一個。

## 9. 加速技巧

1. **Sage Attention** (推薦,明顯加速):
   ```bash
   source ComfyUI/.venv/bin/activate
   pip install sageattention
   deactivate
   ./start.sh --use-sage-attention
   ```
2. **降載模式**:`./start.sh --lowvram`(把部分權重放 CPU/RAM,犧牲速度換 VRAM)。
3. **4-step 閃電工作流**:社群有 distill / lightning 版可把步數降到 4、CFG=1,速度大增。
   - ⚠️ **重要**:網路上常見的 `lightx2v` *high_noise / low_noise* LoRA 是給 **14B 雙專家模型**的,
     **不要**套到 5B 上。5B 請找專屬的「5B 快速 / distill 工作流」。

## 10. 疑難排解

| 症狀 | 解法 |
|---|---|
| `torch.cuda.is_available()` 為 False | 驅動沒裝好或 `nvidia-smi` 失敗 → 重裝驅動並重開機;或 `TORCH_CUDA` 版本與驅動不符,改 `cu124` 重跑 install.sh |
| CUDA OOM (記憶體不足) | 降解析度 → 縮短 frames → `./start.sh --lowvram` |
| 安裝卡在某個 pip 套件 | 重跑 `./install.sh` (idempotent),已裝的會略過,只補失敗的 |
| 模型下載中斷 | 直接重跑 `./download_models.sh`,會自動續傳 |
| torch 安裝失敗、找不到對應輪子 | Ubuntu 26.04 的 3.14 已被 cu128 支援,正常不會發生。若你用很新的 Python (如 3.15) 才會 → 改裝 3.12:`sudo apt install python3.12 python3.12-venv`,刪掉 `ComfyUI/.venv` 後重跑 install.sh |
| 範本裡找不到 Wan2.2 節點 | ComfyUI 太舊 → 重跑 `./install.sh` 會 `git pull` 更新 |
| 輸出全黑 / 雜訊 | 檢查 CFG/Shift,或 VAE 是否載對 (`wan2.2_vae.safetensors`) |
| 速度極慢 | 確認真的在用 GPU (啟動日誌應顯示 CUDA device),裝 Sage Attention |

## 11. 腳本可重複執行的原理

所有腳本都設計成**冪等 (idempotent)**,可以放心重跑:

- **setup.sh**:只是依序呼叫下面兩支,本身無狀態,重跑安全。
- **install.sh**:系統套件用 `dpkg -s` 逐一檢查、只裝缺的,且**會先用 `apt-get install -s` 確認該套件在此 Ubuntu 版本真的裝得起來**(處理像 `libglib2.0-0` 在 24.04/26.04 改名為 `libglib2.0-0t64` 的情況,裝不起來就警告略過而非中斷);ComfyUI 有就 `git pull`;venv 有就重用;PyTorch 用 `torch.version.cuda` 判斷是否已裝 CUDA 版(不靠 `cuda.is_available()`,避免驅動還沒裝時誤判而重裝)。
- **download_models.sh**:每個檔案先比對是否存在且大小達標,完整就略過,未完成就**續傳**(hf/aria2c/wget 三種方式都支援續傳)。

所以安裝中斷、或之後要更新,直接重跑同一個指令(`./setup.sh` 或個別腳本)即可,不會重做已完成的部分。

> 以上行為已在 **Ubuntu 26.04 (x86_64)** 容器中連跑多次實測:套件正確略過、`git pull`/venv 重用、PyTorch 略過分支、模型略過/續傳皆正常。
